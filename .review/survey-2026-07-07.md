# Survey report — `rv` repository (2026-07-07)

Survey pass for an architecture review whose downstream consumer is an agent
reviewing the **documentation site** (https://a2-ai.github.io/rv-docs/, hosted in a
separate repo). This document indexes the codebase so that agent can verify doc
claims against source without prior context. All paths are relative to the repo
root (`/Users/wescummings/projects/rv`). Version surveyed: **0.22.1** (git tag
`v0.22.1`, commit `b9885fa`).

## 1. Repository map

### 1.1 Purpose and type

`rv` is a fast, reproducible, declarative package manager for R, written in Rust.
Users describe a desired project state in an `rproject.toml` config file (R
version, repositories, dependencies); `rv` resolves the dependency tree, records
it in an `rv.lock` lockfile, and installs packages into a project-local library
(`rv/library/`), heavily using a content-addressed cache.

**Codebase type: hybrid CLI + library.** The crate is a Rust library
(`src/lib.rs`) with an optional `cli` feature that gates the `rv` binary
(`Cargo.toml` `[[bin]]` section requires `features = ["cli"]`). The public
library API is the re-export block in `src/lib.rs` (lines 32–66) — external
tooling can embed rv's resolution/sync logic without the CLI. There is no HTTP
server, daemon, or scheduled component.

Sources of stated intent (read these first): `README.md`, `CLAUDE.md` (the most
complete architecture/env-var reference in the repo), `CHANGELOG.md` (detailed,
per-release), `ai-docs/2025-10-18-rhel-research.md` (one research note on RHEL
system deps).

### 1.2 Entry points

- **Binary main:** `src/main.rs` — `fn main()` at line 1306 wraps `try_main()`;
  `Cli::parse()` at line 388; the giant `match cli.command` dispatch begins at
  line 419. This is the only executable entry point.
- **CLI subcommands** (clap derive `enum Command`, `src/main.rs:51` onward):
  `init`, `migrate renv`, `export renv`, `sync`, `add`, `remove`, `upgrade`,
  `plan`, `summary`, `configure repository <add|replace|update|remove|...>`,
  `fmt`, `tree`, `info`, `library`, `cache`, `sysdeps`, `activate`,
  `deactivate`, `run`, `docs`, plus hidden/internal `cli` and `cli-cmds`
  (machine-readable CLI documentation dump — see §1.7).
- **Global flags:** `--config-file/-c` (default `rproject.toml`), verbosity via
  `clap-verbosity-flag`, and an `--emit-events` flag wired to `src/events.rs`.
- **Library entry points:** everything re-exported from `src/lib.rs`, chiefly
  `Context` (`src/context.rs`), `Resolver`/`Resolution` (`src/resolver/`),
  `SyncHandler` (`src/sync/handler.rs`), `Config` (`src/config.rs`).
- **No** HTTP routes, background workers, or cron-style tasks. Long-running work
  is in-process thread pools during `sync` (see §1.6). Ctrl-C handling is
  installed via the `ctrlc` crate (cli feature) feeding `src/cancellation.rs`.

### 1.4 Intended layering

Stated explicitly in `CLAUDE.md` ("Project Structure"):

```
rproject.toml → Config → Resolver → Resolution → SyncHandler → Library
```

Inferred refinement from imports: `main.rs` (cli feature) → `src/cli/*` helpers
→ library API (`Context` construction, then `Resolver`, then `SyncHandler`).
`Context` (`src/context.rs`) is the seam between CLI and library: it loads
config, opens the cache, finds R, and fetches repository databases. The
`cli` feature flag is the enforced layering boundary — library code compiles
without clap/rayon/anyhow (see `#[cfg(feature = "cli")]` guards in
`src/context.rs:4` and `src/sync/handler.rs:11`). Side-effect boundaries are
trait-isolated for testability: `CommandExecutor` for git (`src/git/mod.rs`),
`RCmd` trait for R subprocesses (`src/r_cmd.rs`).

### 1.5 Most load-bearing files

1. **`src/context.rs`** — `Context` is constructed by nearly every subcommand in
   `main.rs`; it owns config, cache, R lookup, and repository databases. Any
   behavior change here touches all commands.
2. **`src/resolver/mod.rs`** (+ `sat.rs`) — the resolution algorithm on the path
   of `plan`, `sync`, `add`, `upgrade`, `tree`, `summary`.
3. **`src/sync/handler.rs`** — `SyncHandler` executes every install/removal;
   owns the concurrency model, cancellation, staging, and the lsof
   in-use-package check.

Honorable mention: `src/config.rs` and `src/consts.rs` — highest fan-in as data
definitions; `consts.rs` is the single source of truth for every env var and
special filename the docs site would document.

### 1.6 Cross-cutting concerns

- **Errors:** per-module `thiserror` types (e.g. `ConfigLoadError`
  `src/config.rs:589`, `LockfileError` `src/lockfile.rs:645`, `SyncErrors`
  `src/sync/errors.rs`, `RCmdError` `src/r_cmd.rs`, plus `src/http.rs`,
  `src/renv.rs`, `src/configure.rs`, `src/run.rs`, `src/activate.rs`,
  `src/sync/link.rs`). CLI layer uses `anyhow`; `main()` prints the error chain
  and exits 1 (`src/main.rs:1306–1311`). This split is a stated convention in
  `CLAUDE.md` ("Code Style Guidelines"). `fs_err` replaces `std::fs` for
  path-annotated IO errors.
- **Config loading:** `src/config.rs` parses `rproject.toml` (serde + toml;
  `toml_edit` for mutations from `add`/`configure`). Env-var configuration is
  centralized in `src/consts.rs` (cache dirs, `RV_LIBRARY_DIR`, `RV_NUM_CPUS`,
  `RV_LINK_MODE` handled in `src/sync/link.rs`, `RV_INSECURE`, etc.);
  `CLAUDE.md` contains full tables that mirror these.
- **Logging:** `log` facade throughout the library; `env_logger` initialized in
  `src/main.rs:406`, level driven by `clap-verbosity-flag`. Progress UX via
  `indicatif` spinners/bars (`src/utils.rs::create_spinner`,
  `src/sync/handler.rs`).
- **Events:** `src/events.rs` — a process-wide `Mutex<Option<Handler>>` that
  serializes typed events to JSON for an external consumer (enabled by the
  `--emit-events` flag; added in the recent "Events (#462)" commit). Task/
  hierarchy model (`Task::child`) lives there too.
- **Auth:** there is none in-process. No Authorization headers or credential
  handling in `src/http.rs` (grep confirms); TLS verification uses ureq's
  platform verifier with an `RV_INSECURE` opt-out (`src/consts.rs:29`). Private
  git dependencies work because git runs as the user's own `git` CLI
  (ssh-agent/credential helpers) — see `src/git/mod.rs:40` and the CI job that
  provisions an SSH key for `example_projects/private-git-dep`
  (`.github/workflows/ci.yaml`, integration job).
- **Validation:** at config-parse time via serde + newtypes (`HttpUrl`,
  `src/config.rs:16`; `Version`/`VersionRequirement`, `src/package/version.rs`);
  lockfile/config consistency checks in `src/lockfile.rs` and the resolver.
- **Concurrency:** `sync` uses `crossbeam` channels + scoped threads with a
  worker pool sized by `RV_NUM_CPUS`/core count (`src/sync/handler.rs`,
  `src/utils.rs::get_max_workers`); shared state via `Arc<Mutex>` and atomics.
  `rayon` (cli-only) parallelizes repository-database fetching in
  `src/context.rs`. Cancellation is a two-stage (soft/hard) atomic counter
  (`src/cancellation.rs`) driven by a `ctrlc` handler; hard cancel can kill R
  processes (`src/r_cmd.rs:207 kill_all_r_processes`). Parallel file
  copy/link with fallbacks in `src/sync/link.rs` (`RV_COPY_THREADS`,
  `RV_LINK_MODE`, network-FS detection via `src/fs.rs::is_network_fs`).
- **Caching:** package databases serialized as MessagePack (`rmp-serde`) with a
  pkgcache-compatible TTL (`PKGCACHE_TIMEOUT`, `src/consts.rs:17–19`); disk
  layout and a multi-user global cache (`RV_GLOBAL_CACHE_DIR`) in
  `src/cache/disk.rs`.

### 1.7 Build / test / CI shape

- **Build:** plain cargo, single crate; `just` recipes wrap common commands
  (`justfile`). `build.rs` embeds a long version string
  (`RV_LONG_VERSION` = version, or version+commit[+timestamp] when untagged/
  dirty). Release profile uses LTO + `codegen-units = 1`.
- **Tests:** inline unit tests in `src/`; integration tests in `tests/`
  (`assert_cmd` + `predicates` drive the real binary; `mockito` fakes repos;
  `rcgen`/`rustls` for TLS tests); `insta` snapshots in `src/*/snapshots/` and
  `tests/snapshots/`; `example_projects/` doubles as an end-to-end fixture set.
- **CI** (`.github/workflows/ci.yaml`): a `test` job (fmt, clippy `-D warnings`,
  `cargo test --all-features -- --include-ignored`) on Linux/macOS/Windows plus
  a check-only musl target, all with **real R 4.5 installed** via
  `r-lib/actions/setup-r`; a separate `integration` job (60-min timeout) that
  provisions an SSH deploy key for the private-git-dep example. Unusual: tests
  require a live R toolchain, and docs/README changes are path-ignored by CI.
- **Release:** `release.yml` + `homebrew.yml` workflows, `rustwrap.yaml`
  packaging config, `scripts/install.sh` curl installer.

  recent notable changes: JSON event emission (#462), no-partial-success for

  docs site is the only user-facing installation/usage reference.
